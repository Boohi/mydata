# mydata v0.1 — Design Spec

**Date:** 2026-05-16
**Status:** Approved (brainstorming phase)
**Next step:** writing-plans skill → implementation plan

## 1. Product

**One-line pitch:** "See where your Mac is sending your data — in real time, by app, by country, by company."

**Problem.** Users have no easy way to see, in everyday terms, where their data is actually going. They might know "Slack uses AWS" but not "Slack sent 2 GB to AWS us-east-1 today and 40 MB to a server in Singapore." Existing tools (LuLu, Little Snitch) are firewalls focused on per-flow allow/block prompts; they don't surface aggregate "where did my data go this week" answers in a way a non-expert can absorb.

**Solution.** A passive, local-only macOS app that captures outbound connection metadata via Apple's NetworkExtension framework, enriches each flow with process attribution, ASN/organization, and country, and presents the result as world maps, sankey flows, app-by-app drilldowns, and timelines.

**Differentiator.** We are the *visualization layer*, not a firewall. v0.1 does not block traffic. Blocking is a deliberate post-v0.1 decision so we do not duplicate LuLu / Little Snitch and so users can run mydata alongside their existing firewall.

**Audience (v0.1).**
- Privacy-conscious Mac users
- Existing LuLu / Little Snitch users curious about aggregate flows
- Journalists and security researchers needing app-level network attribution
- Developers debugging "what does this SDK actually phone home about"

**Reach channels.** Hacker News launch, r/privacy, r/macapps, Objective-See community, privacy newsletters.

## 2. Non-goals (explicitly out of scope for v0.1)

- Blocking traffic
- HTTPS payload inspection / MITM
- Windows or Linux support
- Mobile (iOS / Android)
- Cloud sync, multi-device aggregation
- Browser extension
- Alerts, weekly email digests
- Paid Pro tier
- Distribution to users without macOS Developer Mode willingness for unsigned dev builds (we ship signed + notarized; nothing else)

These get GitHub issues but live in a post-v0.1 milestone so the public roadmap is alive on launch day.

## 3. License, monetization, distribution

- **License:** AGPLv3 for the app and daemon. MIT for any reusable libraries we extract (schema bindings, GeoIP enrichment helpers). AGPL chosen so a competitor cannot SaaS-ify our work without contributing back. We still ship freely distributable binaries.
- **Funding:** GitHub Sponsors. No paid tier in v0.1.
- **Future revenue surface (post-v0.1, not v0.1 scope):** optional paid cloud add-ons (multi-device sync, family/team plans, hosted weekly digests). Open-core, paid is always optional.
- **Distribution:** signed + notarized `.dmg` from GitHub Releases. Apple Developer account is available; Network Extension entitlements will be requested and used legitimately.

### 4.1 Components

**Network extension (`apps/extension`, Swift)**
- One target with both `NEFilterDataProvider` (per-flow metadata, byte counters) and `NEDNSProxyProvider` (DNS query log).
- Emits structured flow events only — no allow/block decisions in v0.1; everything is `.allow`.
- Communicates with the daemon via XPC. Buffers locally if daemon is down.
- Minimal logic. The extension is the most security-sensitive surface; we keep it as small and auditable as possible.

**Daemon (`apps/daemon`, Swift, launchd LaunchAgent)**
- Receives flow + DNS events from the extension via XPC.
- Enriches:
  - Process attribution: `audit_token` → PID → bundle ID, executable path, app icon (cached).
  - Remote endpoint: IP → country (GeoLite2-Country), IP → ASN + org (GeoLite2-ASN), IP → reverse DNS (cached).
  - Joins flows to recent DNS lookups (same process + same destination IP) to attach a hostname when one was resolved.
- Batches and writes to SQLite at `~/Library/Application Support/mydata/events.db`.
- Sweeper task enforces retention window (default 30 days, user-configurable).
- Exposes a read-only XPC endpoint for UI to subscribe to "new flow" events for live updates.

**UI (`apps/ui`, Tauri 2 + React + TypeScript + Vite)**
- Tauri 2 (Rust shell, WKWebView). ~10 MB shell vs ~100 MB Electron.
- Tray icon shows live bytes/sec; click reveals menu (open dashboard, pause, quit).
- Main window views:
  - **Today** — top destinations by bytes, top apps, top countries (cards + bars).
  - **World Map** — choropleth of bytes per country, hover for breakdown (`react-simple-maps` + `visx`).
  - **Apps** — per-app drilldown, sankey of app → destination → country (`visx` sankey).
  - **Timeline** — last 24h byte rate, stacked by app.
  - **Settings** — retention window, exclude list, pause toggle, export DB, about / license / privacy promise.
- Reads SQLite read-only via `tauri-plugin-sql`. Subscribes to daemon XPC for live updates.

### 4.2 Why this split

- The system extension is intentionally tiny so it is easy to audit. The privacy audience will read this code.
- The daemon owns enrichment + storage so the extension stays minimal.
- Tauri's UI is portable: when we later add Windows (WFP) or Linux (eBPF/netfilter) backends, only the extension + daemon swap; the entire React UI is reused.

## 5. Privacy stance (load-bearing)

- **Zero outbound network calls from the app, daemon, or extension.** Every byte of "where does my data go" data stays on the user's Mac.
- **GeoIP / ASN databases are bundled** with the app. Database refreshes ship via app updates, not in-app HTTP fetches.
- **No telemetry, no crash reporting service, no analytics.** None.
- **The privacy promise is enforced by a CI test** (`tests/privacy-paranoia.test.ts` or equivalent) that fails if any shipped binary opens an outbound socket on launch in a sandboxed run.
- **License is AGPLv3** so users can verify these claims by reading the code, and any fork must do the same.

## 6. Data model (sketch — refined in writing-plans)

Tables (SQLite):

- `flows(id, started_at, ended_at, process_id, bundle_id, executable_path, protocol, local_port, remote_ip, remote_port, hostname, bytes_out, bytes_in, country_code, asn, asn_org)`
  - Indexes: `(started_at)`, `(bundle_id, started_at)`, `(country_code, started_at)`, `(asn, started_at)`
- `dns_queries(id, queried_at, process_id, bundle_id, query_name, query_type, resolved_ips)`
  - Indexes: `(queried_at)`, `(query_name)`
- `apps(bundle_id PRIMARY KEY, display_name, icon_blob, first_seen_at, last_seen_at)`
- `exclusions(id, kind, value)` — user-configured exclusions (e.g. LAN subnet, hostname pattern)
- `meta(key PRIMARY KEY, value)` — schema version, retention window, install id (random, never sent)

Schema lives in `packages/schema`. TS types and Swift structs are generated from the same source-of-truth file so drift is impossible.

## 7. Testing strategy

- **Swift extension:** unit tests via `swift test` for event parsing, byte accounting, XPC framing.
- **Daemon:** integration tests against in-memory SQLite; golden-file tests for enrichment outputs given fixed IP fixtures.
- **UI:** Vitest + React Testing Library for components; Playwright for onboarding + main-window smoke.
- **End-to-end:** a "loopback talker" test harness opens known connections (HTTP to a fixture server, DNS lookups for known hostnames) and asserts they appear in the SQLite with correct attribution within N seconds.
- **Privacy paranoia test:** see §5. Runs in CI on every PR.
- **Manual QA checklist** in `docs/qa/release-checklist.md` for items we cannot automate (Network Extension approval prompt, notarization, .dmg open flow).

## 10. Risks and open questions

- **Network Extension approval friction.** First-launch UX requires the user to approve a system extension in System Settings. Onboarding must walk through this. Mitigation: dedicated onboarding screen with screenshots; LuLu has solved this UX and we can reference their pattern.
- **GeoIP licensing.** MaxMind GeoLite2 is free but requires attribution and a license agreement. DB-IP Lite is CC-BY-4.0. Decision deferred to writing-plans; default to DB-IP Lite for simpler redistribution under AGPL.
- **Process attribution edge cases.** Short-lived processes may exit before the daemon resolves their bundle ID. Mitigation: capture `audit_token` in the extension synchronously, resolve in the daemon, store raw token if resolution fails.
- **Notarization in CI.** Requires Apple credentials in GitHub Actions secrets. Mitigation: standard `notarytool` flow; document the secret setup in `docs/build.md`.
- **Tauri 2 maturity on macOS.** Tauri 2 is stable but newer than Electron; we accept this trade for install size and Rust safety.

## 11. Success criteria for v0.1

- A user can download the `.dmg` from GitHub Releases, drag to Applications, double-click, approve the system extension, and within 60 seconds see real flows in the Today view.
- All four views (Today / Map / Apps / Timeline) render meaningful data for a normal day's macOS usage.
- Privacy paranoia test passes in CI.
- Source builds cleanly on a fresh Mac following `docs/build.md`.
- README has a working animated demo gif.
- HN launch post drafted and ready.
